## 1. 이 문서의 목적

이 문서는 **클라이언트가 제공한 레퍼런스 웹사이트를  
‘따라하지 않고 활용하기 위한’ 해석 규칙**을 정의한다.

레퍼런스는:
- 복사 대상 ❌
- 스타일 모음 ❌
- 트렌드 샘플 ❌  

→ **구조적 사고 재료**다.

---

## 2. 레퍼런스의 정의 재설정

### 2.1 레퍼런스란 무엇인가

레퍼런스는  
“저 브랜드처럼 보이고 싶다”가 아니라,

> “저 브랜드는  
> 어떤 방식으로 인식을 설계했는가?”를 묻기 위한 대상이다.

---

## 3. 레퍼런스 해석의 3단계 필터

모든 레퍼런스는 아래 3단계를 거쳐야만 한다.

---

### 3.1 Layer 1 — Surface (무시 대상)

아래 요소들은 **직접 차용 금지**:

- 색상
- 폰트
- 그래픽 스타일
- 3D / 영상 연출
- 특정 인터랙션 효과

> Surface는 **결과**이지 **의도**가 아니다.

---

### 3.2 Layer 2 — Structure (분석 대상)

집중 분석 대상:

- 첫 화면에서 무엇을 말하지 않는가
- 정보의 등장 순서
- 밀도 변화 구간
- 시선의 멈춤 지점
- 반복되는 섹션 리듬

→ 이 레이어가 **가져올 수 있는 전부**다.

---

### 3.3 Layer 3 — Intention (핵심)

반드시 도출해야 하는 질문:

- 이 페이지는 무엇을 증명하려 하는가?
- 무엇을 숨기고 무엇을 늦추는가?
- 사용자가 언제 “아, 알겠다”라고 느끼는가?

---

## 4. 레퍼런스 분해 템플릿

모든 레퍼런스는 아래 항목으로만 기록한다.

### 4.1 레퍼런스 기록 포맷 (필수)

각 레퍼런스는 아래 형식으로만 정리한다.

- Reference Name:
- Reference URL:
- 관찰된 핵심 구조:
- 정보 공개 리듬:
- 감정 전환 지점:
- 차용 가능 요소 (Structure / Logic only):
- 차용 금지 요소 (Surface):

이미지, 색, 그래픽, 톤에 대한 평가는 기록하지 않는다.

## 5. 클라이언트 레퍼런스 적용 원칙 (KOLLAB 전용)

본 프로젝트에서 제공된 레퍼런스(갤럭시, 이노레드 등)는  
**아트워크 및 레이아웃 감각 참고용**으로만 사용한다.

적용 원칙은 다음과 같다.

1. 레퍼런스는 결과물이 아닌 **사고 방식**만 추출한다.
2. 페이지 구조는 단순화된 블록 시스템으로 재구성한다.
3. 하나의 레퍼런스에서 하나의 인사이트만 취한다.
4. 복수 레퍼런스를 혼합하여 새로운 구조를 만든다.
5. 최종 결과물은 레퍼런스를 연상시켜서는 안 된다.

이 규칙은 이후 모든 브랜드 페이지 프로젝트에도 그대로 적용된다.
