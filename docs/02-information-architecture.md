## 1. 이 문서의 목적

이 문서는 **Brand Page의 정보 구조(IA)를 설계하는 절대 규칙**을 정의한다.  
디자인 이전, 콘텐츠 이전에 **정보가 어떤 논리로 배치되어야 하는지**를 결정한다.

이 문서는:
- 화면 설계서 ❌
- 와이어프레임 ❌  
→ **“무엇이 먼저 인식되어야 하는가”에 대한 설계 문서**다.

- 회사소개 순서 ❌

---

## 3. Brand Page IA의 4단 구조

이 4단 구조는 **레이아웃 이전의 사고 구조**이며,  
레퍼런스와 무관하게 항상 유지되어야 한다.

```
1. Signal
2. Context
3. Evidence
4. Continuity
```

각 단계는 독립적이며,  
하나가 빠지면 Brand Page는 성립하지 않는다.

---

## 5. 단일 페이지 vs 멀티 페이지 판단 기준

### 5.3 단순 레이아웃 우선 원칙 (Client Alignment)

본 프로젝트(KOLLAB)는 다음 원칙을 따른다.

- 복잡한 내비게이션 구조를 사용하지 않는다
- 사용자는 스크롤만으로 전체 인식을 형성해야 한다
- 멀티 페이지 구조라도 IA는 단일 서사처럼 작동해야 한다

이는 속도, 명확성, 반복 제작을 우선하기 위한 결정이며  
브랜드 수준을 낮추기 위한 선택이 아니다.

## 10. IA 설계 검증 질문 (Client-facing)

다음 질문에 모두 “Yes”일 때만 IA를 확정한다.

- 이 구조는 1주일 안에 구현 가능한가?
- 레이아웃 설명 없이도 의도가 전달되는가?
- 불필요한 섹션을 제거했는가?
